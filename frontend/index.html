SQL Injection

SQL Injection Tutorial

SQL Injection is a code injection technique that exploits vulnerabilities in applications that interact with databases. This attack occurs when user input is incorrectly filtered and directly included in SQL queries.

How to Test:

  1. Enter the target application URL
  2. Enter ' OR '1'='1 in the username field
  3. Enter anything in the password field
  4. Submit the form

Why it works:

The application builds an SQL query by directly concatenating user input. When you enter the payload, the query becomes:

SELECT * FROM users WHERE username = '' OR '1'='1' AND password = 'anything'

Since '1'='1' is always true, this bypasses authentication.

Prevention methods:

  • Use parameterized queries or prepared statements
  • Implement input validation and sanitization
  • Apply the principle of least privilege for database accounts
  • Use ORM frameworks that handle SQL escaping automatically

Cross-Site Scripting (XSS)

XSS Tutorial

Cross-Site Scripting (XSS) is a client-side code injection attack where attackers inject malicious scripts into websites viewed by users. These scripts execute in the victim's browser and can steal cookies, session tokens, or other sensitive information.

Types of XSS:

  • Reflected XSS: Malicious script comes from the current HTTP request
  • Stored XSS: Malicious script is stored on the target server
  • DOM-based XSS: Vulnerability exists in client-side code

How to Test:

  1. Enter the target application URL
  2. Enter <script>alert('XSS')</script> in the input field
  3. Submit the form
  4. If an alert box appears, the application is vulnerable

Prevention methods:

  • Encode output data (HTML, URL, JavaScript, CSS encoding)
  • Implement Content Security Policy (CSP)
  • Use frameworks that automatically escape XSS
  • Sanitize user input
  • Set HttpOnly flag on cookies

Brute Force Attack

Brute Force Tutorial

Brute force attacks attempt to gain access to accounts by systematically trying all possible password combinations until the correct one is found. These attacks exploit weak passwords and lack of account lockout mechanisms.

Types of Brute Force Attacks:

  • Simple Brute Force: Trying every possible combination
  • Dictionary Attack: Using a list of common passwords
  • Credential Stuffing: Using leaked username/password pairs
  • Password Spraying: Using few common passwords across many accounts

How to Test:

  1. Enter the target application URL
  2. Enter a username to target
  3. Enter a list of passwords to try (one per line)
  4. Click "Start Attack" to begin the brute force attempt

Prevention methods:

  • Implement account lockout after failed attempts
  • Use CAPTCHA or other human verification
  • Enforce strong password policies
  • Implement multi-factor authentication
  • Add time delays between login attempts

Path Traversal

Path Traversal Tutorial

Path traversal (also known as directory traversal) is an attack that allows attackers to access files and directories outside the intended directory by manipulating file paths. This vulnerability can expose sensitive system files.

How to Test:

  1. Enter the target application URL
  2. Enter a path traversal payload like ../../../etc/passwd
  3. Submit the request
  4. If you can view files outside the intended directory, the application is vulnerable

Common Payloads:

  • ../../../etc/passwd (Unix/Linux)
  • ..\..\Windows\win.ini (Windows)
  • ....//....//....//etc/passwd (Filter bypass)
  • %2e%2e%2f%2e%2e%2f%2e%2e%2fetc%2fpasswd (URL encoded)

Prevention methods:

  • Validate and sanitize user input
  • Use proper file access controls
  • Implement whitelisting for allowed files
  • Avoid passing user-supplied input to filesystem APIs
  • Use a secure file access library

Command Injection

Command Injection Tutorial

Command injection is an attack where attackers execute arbitrary commands on the host operating system through a vulnerable application. This occurs when applications pass unsafe user-supplied data to a system shell.

How to Test:

  1. Enter the target application URL
  2. Enter a command injection payload like 127.0.0.1; cat /etc/passwd
  3. Submit the request
  4. If the command executes, the application is vulnerable

Common Payloads:

  • ; cat /etc/passwd (Unix/Linux)
  • & dir (Windows)
  • | whoami (Pipe operator)
  • $(cat /etc/passwd) (Command substitution)
  • `cat /etc/passwd` (Backtick substitution)

Prevention methods:

  • Avoid using system commands with user input
  • Use safer APIs that don't invoke the shell
  • Implement strict input validation
  • Use allowlists for permitted values
  • Run applications with minimal privileges
