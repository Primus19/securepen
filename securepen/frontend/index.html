SQL Injection

ℹ️

SQL Injection Tutorial

SQL Injection is a code injection technique that exploits vulnerabilities in applications that interact with databases.

How to Test:
  1. Enter ' OR '1'='1 in the username field
  2. Enter anything in the password field
  3. Submit the form
Why it works:

The application builds an SQL query by directly concatenating user input. When you enter the payload, the query becomes:

SELECT * FROM users WHERE username = '' OR '1'='1' AND password = 'anything'

Since '1'='1' is always true, this bypasses authentication.

Real-world impact:

Attackers can bypass authentication, extract sensitive data, modify database content, or even delete entire tables.

Try: ' OR '1'='1

Cross-Site Scripting (XSS)

ℹ️

XSS Attack Tutorial

Cross-Site Scripting allows attackers to inject client-side scripts into web pages viewed by other users.

How to Test:
  1. Enter <script>alert('XSS')</script> in the comment field
  2. Submit the form
  3. The script will execute, showing an alert
Alternative payloads:
  • <img src="x" onerror="alert('XSS')">
  • <body onload="alert('XSS')">
  • javascript:alert('XSS')
Real-world impact:

Attackers can steal cookies, session tokens, redirect users to malicious sites, or perform actions on behalf of the victim.

Vulnerable Comment

Try: <script>alert('XSS')</script>

Brute Force

ℹ️

Brute Force Attack Tutorial

Brute force attacks attempt to crack passwords by trying many combinations until finding the correct one.

How to Test:
  1. Enter a username (try "admin")
  2. Enter a list of passwords (one per line)
  3. Include "password123" in your list
  4. Submit the form to attempt to crack the password
Sample wordlist:
123456
password
admin
qwerty
welcome
password123
letmein
Real-world impact:

Attackers can gain unauthorized access to accounts, especially those with weak passwords or without proper protections.

Path Traversal

ℹ️

Path Traversal Tutorial

Path traversal (directory traversal) attacks allow attackers to access files and directories outside the intended directory.

How to Test:
  1. Enter ../../../etc/passwd in the filename field
  2. Try variations like ..%2f..%2f..%2fetc%2fpasswd
  3. Submit to attempt to access files outside the web root
Real-world impact:

Attackers can access sensitive configuration files, source code, or other critical system files not intended for public access.

Command Injection

ℹ️

Command Injection Tutorial

Command injection attacks allow execution of arbitrary commands on the host operating system.

How to Test:
  1. Enter a hostname like localhost
  2. Then try localhost; ls -la
  3. Or localhost && cat /etc/passwd
  4. Submit to see if commands are executed
Real-world impact:

Attackers can execute system commands to access sensitive data, modify files, or even gain complete control of the server.
